KEY FINDINGS (briefing)
- Toolchain consensus is settled: Ollama + LM Studio as entry points, Open WebUI for local-doc RAG, Continue.dev for local code autocomplete, OpenClaw for local agents.
- Biggest disagreement: capability vs cloud. Tina Huang / Daniel Bourke say open models are now "just as good" (4B models beating GPT-4o on benchmarks, running on an iPhone); Zen van Riel / lustoykov say there's still a "huge gap" for frontier reasoning and vendor charts are cherrypicked. Pragmatic middle ground (Rynaut): a hybrid 80/20 split - local for routine/private tasks, cloud for the hard 20%.
- Hardware: memory is the binding constraint. NVIDIA VRAM spillover collapses throughput (~40 -> 2-3 tok/s); Apple unified memory is the disruptor (128GB Mac Studio runs a 120B model); M5 Max ~4x prompt processing; 4-bit quantization cuts memory ~75% for <2% quality loss.
- Privacy is universally praised, but "local = safer" is contested: a misconfigured local agent with terminal/filesystem access is a prompt-injection backdoor; van Riel even prefers cloud models to drive local agents.
- Hype vs substance: working well today = speech-to-text/TTS, image generation, code autocomplete. Overhyped = autonomous local "vibe coding", local video generation, long voice agents.
- Honesty note: the notebook flagged that the "BitNet 1-bit LLM" source (Rynaut, 107 subs) does not actually discuss BitNet in its transcript - that material came from outside the sources. Kept this caveat in the report.
